# API Documentation

The app uses two complementary server interfaces:

1. **Route handlers** (`src/app/api/*`) for authentication, registration, and AI generation.
2. **Server Actions** (`src/lib/actions.ts`) for all data mutations (trips, cities, days, activities, expenses, packing, checklist, notes, favorites, admin). Server Actions are invoked directly from forms and client components and are not public REST endpoints.

This document covers the HTTP route handlers and summarizes the server actions.

## Authentication model

- Sessions are JWT based (Auth.js v5, Credentials provider).
- Protected pages are gated by `src/middleware.ts` (cookie check) and enforced in each server component via `requireUser()` / `requireAdmin()`.
- Authorization for trip data is enforced by `getOwnedTrip()`, which returns a trip only if the caller owns it or is an admin.

## Route handlers

### POST `/api/register`

Create a new user account.

Request body:
```json
{ "name": "Jane", "email": "jane@example.com", "password": "secret123" }
```

Responses:
- `201` `{ "ok": true }`
- `400` `{ "error": "<validation message>" }`
- `409` `{ "error": "An account with this email already exists" }`
- `500` `{ "error": "Something went wrong. Please try again." }`

Validation (zod): name 2-60 chars, valid email, password 6-100 chars.

### POST `/api/generate`

Generate a trip itinerary. Requires an authenticated session.

Request body:
```json
{
  "destination": "Tokyo, Japan",
  "days": 5,
  "interests": "food, temples, nightlife",
  "pace": "balanced",
  "budgetLevel": "mid"
}
```

Field constraints:
- `destination`: string, 2-120 chars (required)
- `days`: integer, 1-21 (required)
- `interests`: string, up to 300 chars (optional)
- `pace`: `relaxed` | `balanced` | `packed` (default `balanced`)
- `budgetLevel`: `budget` | `mid` | `luxury` (default `mid`)

Response `200`:
```json
{
  "destination": "Tokyo, Japan",
  "source": "openai",
  "days": [
    {
      "dayNumber": 1,
      "city": "Tokyo, Japan",
      "summary": "Arrival and first taste of Tokyo.",
      "activities": [
        {
          "time": "12:00",
          "title": "Arrive in Tokyo & hotel check-in",
          "description": "Settle in and get oriented.",
          "location": "Tokyo, Japan",
          "type": "HOTEL",
          "cost": 0
        }
      ]
    }
  ]
}
```

`source` is `"openai"` when generated by the OpenAI API, or `"engine"` when produced by the built-in deterministic engine (used when `OPENAI_API_KEY` is unset or the API call fails).

Error responses:
- `401` `{ "error": "Unauthorized" }`
- `400` `{ "error": "<validation message>" }`
- `500` `{ "error": "Failed to generate itinerary" }`

### `GET|POST /api/auth/[...nextauth]`

Auth.js endpoints (sign in, sign out, session, CSRF). Managed by Auth.js; not called directly by application code.

## Server Actions (summary)

Defined in `src/lib/actions.ts`. Each mutation validates ownership before writing and calls `revalidatePath` to refresh the affected pages.

| Action                        | Purpose                                  |
| ----------------------------- | ---------------------------------------- |
| `createTrip` / `updateTrip` / `deleteTrip` | Trip lifecycle              |
| `toggleTripPublic`            | Publish or unpublish the share page      |
| `addCity` / `deleteCity`      | Multi-city stops                         |
| `saveGeneratedItinerary`      | Persist an AI-generated itinerary        |
| `addDay` / `deleteDay`        | Itinerary days                           |
| `addActivity` / `deleteActivity` | Day activities                        |
| `addExpense` / `deleteExpense`| Expense tracker                          |
| `addPackingItem` / `togglePackingItem` / `deletePackingItem` / `generatePackingList` | Packing list |
| `addChecklistItem` / `toggleChecklistItem` / `deleteChecklistItem` / `generateChecklist` | Checklist |
| `addNote` / `deleteNote`      | Hotel/flight/location notes              |
| `addFavorite` / `deleteFavorite` | Favorite destinations                 |
| `adminDeleteUser` / `adminSetRole` | Admin user management (admin only)  |
